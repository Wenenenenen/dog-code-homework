# 展会摊位排期与冲突检查系统

一个用于模拟展会主办方给不同参展商安排摊位和时间段的管理系统，支持实时冲突检测和报告生成。

## 功能特性

### 核心功能
- **排期管理**：支持新增、编辑、删除排期记录
- **多视图展示**：支持表格视图和看板视图两种展示方式
- **多维度查看**：支持按区域、时间段、参展商三种维度查看排期
- **实时冲突检测**：自动检测并标记两种类型的冲突
  - 摊位冲突：同一摊位在同一时间段被重复占用
  - 参展商冲突：同一参展商在同一时间段被重复安排
- **本地数据持久化**：所有数据保存到浏览器 localStorage，刷新页面后仍然保留
- **冲突报告生成**：一键生成详细的冲突报告，包含冲突详情和处理建议

### 辅助功能
- **数据导入导出**：支持JSON格式的数据导入和导出
- **数据重置**：可一键重置为默认示例数据
- **通知系统**：操作成功/失败的实时通知提示
- **响应式设计**：适配桌面端和移动端

## 关键设计思路

### 1. 数据模型设计

系统包含五类核心数据实体：

- **区域（Area）**：展会的不同展区（如A区科技展区、B区消费品展区等）
- **摊位（Booth）**：每个区域内的具体展位，包含编号、大小、位置描述
- **参展商（Exhibitor）**：参与展会的公司/机构信息
- **时间段（TimeSlot）**：展会的时间分段（按天的上午/下午划分）
- **排期（Schedule）**：将参展商、摊位、时间段关联起来的记录

### 2. 冲突检测算法

冲突检查核心逻辑位于 `src/utils/conflictChecker.js`：

**检测规则：**
1. **摊位冲突**：两条排期记录具有相同的 `boothId` 和 `timeSlotId`
2. **参展商冲突**：两条排期记录具有相同的 `exhibitorId` 和 `timeSlotId`

**算法优化：**
- 按时间段分组检测，减少不必要的比较
- 去重处理，避免同一对冲突被重复报告
- 提供冲突类型分类和处理建议

### 3. 状态管理模式

采用发布-订阅模式实现状态管理：

```javascript
// 订阅状态变化
const unsubscribe = subscribe(newState => {
  // 更新UI
});

// 状态变更时自动通知所有订阅者
notifyListeners();
```

### 4. 本地存储策略

- 使用浏览器 localStorage 持久化存储
- 数据变更后自动保存（增删改操作）
- 页面加载时自动从存储恢复数据
- 支持JSON格式的导入导出

### 5. 组件架构

采用组件化设计，每个组件职责单一：

- **App.js**：应用入口，统一管理视图渲染和事件绑定
- **TableView.js**：表格视图，按不同维度分组展示
- **BoardView.js**：看板视图，以卡片形式展示排期
- **ScheduleModal.js**：排期编辑表单
- **ConflictReport.js**：冲突报告展示
- **Notifications.js**：通知提示

## 验证步骤

### 1. 基础功能验证

1. **启动应用**
   - 执行 `npm install` 安装依赖
   - 执行 `npm run dev` 启动开发服务器
   - 确认浏览器自动打开并显示应用界面

2. **查看初始数据**
   - 确认页面显示示例排期数据
   - 确认顶部显示冲突警告（示例数据包含冲突）

### 2. 视图切换验证

1. **表格/看板视图切换**
   - 点击「表格」按钮，确认显示表格视图
   - 点击「看板」按钮，确认显示看板视图

2. **维度切换验证**
   - 点击「按区域」，确认排期按区域分组
   - 点击「按时间段」，确认排期按时间段分组
   - 点击「按参展商」，确认排期按参展商分组

### 3. 冲突检测验证

1. **查看现有冲突**
   - 确认冲突行/卡片有明显的视觉标记（红色背景、冲突徽章）
   - 点击「生成冲突报告」按钮，查看详细冲突信息

2. **创建新冲突**
   - 点击「新增排期」按钮
   - 选择一个已被占用的摊位和相同时间段
   - 保存后，确认系统显示冲突警告通知
   - 确认新排期被标记为冲突状态

### 4. 排期管理验证

1. **新增排期**
   - 点击「新增排期」
   - 填写完整信息（选择未冲突的摊位、参展商、时间段）
   - 保存后，确认新排期出现在列表中
   - 确认显示「排期创建成功」通知

2. **编辑排期**
   - 点击任意排期的「编辑」按钮
   - 修改部分信息（如备注）
   - 保存后，确认修改已生效
   - 确认显示「排期更新成功」通知

3. **删除排期**
   - 点击任意排期的「删除」按钮
   - 确认删除操作
   - 确认排期从列表中移除
   - 确认显示「排期删除成功」通知

### 5. 数据持久化验证

1. **刷新页面**
   - 记录当前页面的排期状态
   - 刷新浏览器页面
   - 确认所有数据保持不变

2. **数据导出**
   - 点击「导出数据」按钮
   - 确认下载了JSON文件
   - 检查文件内容是否包含所有数据

3. **数据重置**
   - 点击「重置数据」按钮
   - 确认重置操作
   - 确认数据恢复为初始示例状态

### 6. 响应式验证

1. **桌面端**
   - 确认所有元素正常显示，无布局问题
   - 确认表格可以横向滚动（如需要）

2. **移动端**
   - 使用浏览器开发者工具切换到移动端视图
   - 确认布局自动调整
   - 确认看板视图变为垂直布局
   - 确认所有按钮可点击

## 开发说明

### 添加新的模拟数据
编辑 `src/data/mockData.js` 文件，修改对应的数据数组。

### 自定义样式
样式文件位于 `src/styles/main.css`，使用 CSS 变量定义主题色，可通过修改 `:root` 中的变量快速调整整体风格。

### 扩展功能
- 新增数据实体：在 `src/data/mockData.js` 添加数据，在 `src/models/scheduleModel.js` 添加验证逻辑
- 新增视图：在 `src/components/` 目录下创建新组件，在 `App.js` 中集成
- 新增冲突类型：在 `src/utils/conflictChecker.js` 中扩展检测逻辑
